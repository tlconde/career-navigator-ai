## Design Context

### Users

**CareerBridge** serves **refugees and newcomers** navigating job search in a new context, plus anyone who wants browser-based career coaching without CLI or Git. Primary job-to-be-done: get interview-ready, improve CVs for ATS, evaluate roles, and access coaching tips—**in the browser**, with optional language choice. Advanced tools mirror [career-ops](https://github.com/santifer/career-ops) workflows for power users; heavy automation stays local in career-ops. Emotional framing in-product is **warm and trustworthy** (hero tagline, heart/shield motifs, privacy banner).

### Brand Personality

**Voice:** Clear, supportive, non-technical for core flows; respectful of stress and uncertainty in job search.  
**Three words:** *Warm · trustworthy · approachable* (inferred from README positioning, UI copy patterns, and palette).  
**Goals:** Confidence and clarity—not hype or gamification. Credit and methodology transparency (career-ops link in footer).

### Aesthetic Direction

**Visual tone:** Warm, soft neutrals (cream/warm gray backgrounds via HSL `--background`), **teal-green primary** (`--primary` ~152°) and **warm orange secondary/accent** (~32°), rounded corners (`--radius` 0.75rem), generous whitespace.  
**Typography:** **Nunito** for headings (bold/extrabold), **Inter** for body—friendly but readable.  
**Stack (implementation):** Vite 5, React 18, TypeScript, **Tailwind CSS** + **shadcn/ui** (Radix primitives), **Lucide** icons, **React Router**, **TanStack Query**, **i18next** (EN, AR RTL, UK, FR, ES, TR), toasts via Sonner + shadcn Toaster. Layout: `max-w-6xl` header/footer, sticky blurred header, card-based sections.  
**Theme:** Light mode is the default experience; **dark** CSS variables exist (`tailwind` `darkMode: class`) but the app is not wrapped in a global theme provider in `main`/`App`—treat dark as **supported in tokens**, not yet a first-class user toggle unless added.  
**References / anti-references:** Not specified in-repo—see open questions below.

### Design Principles

1. **Accessibility and plain language first** — No assumption of technical literacy on main paths; labels and errors should stay understandable; RTL and non-English flows are first-class.
2. **Warm minimalism** — Use the existing token system (primary/secondary/muted/card); avoid cold “default dashboard” gray or noisy decoration; favor calm hierarchy over density.
3. **Trust through transparency** — Privacy and data handling (local storage, edge AI) should remain visible where the product already surfaces them; avoid dark patterns.
4. **Consistent with shadcn patterns** — Prefer extending `components/ui` and shared layout (`Layout`, cards, buttons) over one-off styling.
5. **Motion with restraint** — Radix/shadcn defaults (e.g. accordion) are fine; avoid gratuitous animation that could distract or trigger vestibular issues (align with any formal a11y bar once confirmed).
